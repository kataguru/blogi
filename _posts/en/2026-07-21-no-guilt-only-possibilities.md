I have a lot of unfinished projects on my computer: books, software, plans, notes and thoughts that were never finished in their original form.

Previously, an unfinished project easily felt like an open task. If I had started something, it should also be finished sometime — otherwise the project would remain in the background as a reminder of the unfinished work. This turned the warehouse into a to-do list, and each folder contained not only an opportunity, but also a little guilt.

Now I understand it differently.

**Moving a project to storage does not mean an obligation to use it. It only preserves the possibility.**

## Inventory is not a to-do list

An item on the to-do list is waiting to be completed; the item in stock is waiting for nothing. It has no deadline, no next task, and no promise to come back to it again. It just exists.

The project can later be reopened if a suitable purpose arises for it. One thought, image, piece of text or technical solution can be taken from it — or it will never be used. That is also a completely acceptable result.

The purpose of the warehouse is not to create future work, but to prevent unnecessary loss of work already done.

## Opportunity without debt

In economics, an option means the right to do something without the obligation to do it. The same idea also works in creative work. When I move a project to storage, I keep the ability to:

* continue it later
* merges it with another project
* break it down into usable parts
* uses its ideas in a new context
* leaves it permanently alone

None of these are mandatory. A project can have option value even if it is never activated. Just knowing that the material is stored is enough: I don't need to decide now whether the project will be useful five years from now. I can leave the decision to my future self.

## Incompleteness only consumes if it remains active

An unfinished project burdens you when it remains in active memory and reminds you: you should continue, you should decide, you should finish, you should at least clean the files. When enough projects accumulate, each of them takes a small part of the attention. A single open loop doesn't seem like much, but together they form a continuous background load.

Passivating the project breaks this connection. It is no longer a task in progress but a stored material. The difference is small in words but big in experience: an unfinished project requires a solution, a stored project does not require anything.

## You don't have to use everything

Recycling can also be made a performance requirement. If I have saved the old text, I might have thought that I should find a use for it someday — otherwise it would have been pointless to keep it. But this idea brings the duty back.

The value of the inventory does not depend on how much of its content is used. Even in a toolbox, not every tool needs to be in constant use; its value comes from having the right tool available when you need it. The same applies to ideas and projects: some of them later turn out to be valuable, some remain permanently unused. There is no need to make a final choice between them in advance.

## Retention is not a promise

This is the most important realization for me. When I keep a project, I don't promise it a future. I don't promise to finish it, I don't promise to publish it, I don't even promise to reopen it. Retention just means I don't destroy the opportunity unnecessarily.

At the same time, I can release the project in my mind. It doesn't have to come with you as an obligation and it doesn't have to remind you of what wasn't done. It can exist without demands.

## A creative warehouse is a reserve of possibilities

The work done over the years gradually forms a reserve of possibilities. There are ideas the meaning of which I do not yet understand, solutions to problems I have not yet encountered, and texts that did not fit their original purpose but may later find the right environment.

Inventory grows with each project, but it doesn't have to be a burden. In AFRS thinking, old work can be recycled into components of new projects; my new realization is that recycling is always voluntary.

The component warehouse is not a backlog or a list of jobs that are waiting for their author. It's a set of possibilities, none of which you have to use.

There is no guilt involved. There are only possibilities.
